## 1. HCL syntax primer

HCL is what Terraform / OpenTofu consumes. Mental model: yaml is data; HCL is data + references + a tiny expression language. Same nested-key feel; different punctuation.

**Blocks** — top-level units. Shape: `type "label1" "label2" { ... }`. For `resource` blocks, label1 is the provider-defined type, label2 is your local name.

```hcl
resource "aws_s3_bucket" "uploads" {       # type=aws_s3_bucket, local name=uploads
  bucket = "shoplet-uploads-staging"       # key = value (note: = not :)
  tags   = { Env = "staging" }             # nested object literal
}
```

**Attributes** — `key = expression` inside a block. Strings in `"..."`; numbers, booleans, lists `[...]`, objects `{...}` look like JSON.

**References are first-class** — there are no string-interpolated IDs. Define `aws_s3_bucket.uploads`, and anywhere else `aws_s3_bucket.uploads.arn` reads that resource's ARN attribute, automatically wiring a dependency edge in Terraform's graph.

```hcl
resource "aws_iam_role_policy" "api_uploads" {
  role   = aws_iam_role.api.id                            # reference, not a string
  policy = data.aws_iam_policy_document.uploads.json
}
```

String interpolation when needed: `"arn:aws:s3:::${aws_s3_bucket.uploads.id}/*"`.

**Heredocs** — multi-line strings (JSON policies, user-data scripts). `<<-` strips leading whitespace:

```hcl
policy = <<-EOF
  { "Statement": [{"Effect": "Allow", "Action": "s3:GetObject"}] }
EOF
```

**Variables** — inputs. `variable "x" { ... }` declares, `var.x` reads:

```hcl
variable "region" { type = string; default = "us-east-1" }
provider "aws" { region = var.region }
```

**`for_each`** — loops a block over a map. `each.key` and `each.value` reference the current pair:

```hcl
resource "aws_sqs_queue" "q" {
  for_each = { jobs = "standard", events = "fifo" }
  name     = "${each.key}-staging"
}
# Reference one: aws_sqs_queue.q["jobs"].arn
```

**`dynamic`** — generates repeated nested blocks (e.g. multiple `ingress {}` inside one SG):

```hcl
dynamic "ingress" {
  for_each = var.ports
  content {
    from_port = ingress.value
    to_port   = ingress.value
    protocol  = "tcp"
  }
}
```

**Modules** — reusable HCL bundles (`module "x" { source = "..."; ... }`). Supeux's v1 emit is flat files (no `module {}` blocks) for legibility.

`jsonencode(...)` and `tomap(...)` are common helpers — they convert HCL values to JSON strings / typed maps for fields that demand them (IAM policies, ECS container definitions).

That's enough to read everything below.

## 2. Worked HCL for `staging-fargate`

Files supeux emits into `infra/staging-fargate/generated/`. Driven by the yaml in [ir.md §2](ir.md#2-yaml-schema-worked-example).

### 2a. `main.tf`

```hcl
# === Generated by supeux compile --target=staging-fargate. DO NOT EDIT. ===
# Hand-corrections belong in sibling .tf files (infra/staging-fargate/*.tf).

# ----- resources.uploads (bucket / standard) -----
resource "aws_s3_bucket" "uploads" {
  bucket = "shoplet-uploads-staging-fargate"            # <app>-<resource>-<target>
}
# class:standard → no extra storage-class config (S3 default).
# Glacier/IA classes would add an aws_s3_bucket_lifecycle_configuration here.

# ----- resources.jobs (queue / standard) -----
resource "aws_sqs_queue" "jobs" {
  name                       = "shoplet-jobs-staging-fargate"
  visibility_timeout_seconds = 30                        # supeux default; override via yaml `visibility:`
  message_retention_seconds  = 345600                    # 4 days, AWS default
}

# ----- resources.app_db (db.sql / postgres-16 / tier=dev) -----
# Tier table (supeux-owned):
#   dev        → db.t3.micro,  20 GB,  single-AZ, no backup     ← this one (target override)
#   prod-small → db.t3.medium, 100 GB, single-AZ, 7-day backup
#   prod       → db.m6g.large, 200 GB, multi-AZ,  14-day backup
#   premium    → db.r6g.xlarge, 500 GB, multi-AZ, 35-day backup
#   global     → Aurora Global Database (different resource shape)
resource "aws_db_subnet_group" "app_db" {
  name       = "shoplet-app-db-staging-fargate"
  subnet_ids = [for s in aws_subnet.private : s.id]
}

resource "aws_db_instance" "app_db" {
  identifier                  = "shoplet-app-db-staging-fargate"
  engine                      = "postgres"
  engine_version              = "16"
  instance_class              = "db.t3.micro"            # from tier=dev
  allocated_storage           = 20
  multi_az                    = false
  db_name                     = "app"
  username                    = "app"
  manage_master_user_password = true                     # AWS-managed credential in Secrets Manager
  db_subnet_group_name        = aws_db_subnet_group.app_db.name
  vpc_security_group_ids      = [aws_security_group.app_db.id]
  backup_retention_period     = 0
  skip_final_snapshot         = true
}

# ----- ECR repo (single image, multi-entrypoint) -----
# Yaml has separate `build:` paths per module, but the EMITTED image is ONE: built
# from repo root, dispatcher selects which module by --module=<name> CLI arg in the
# task definition. Modular-monolith bundle would pass --modules=a,b,c.
resource "aws_ecr_repository" "app" {
  name                 = "shoplet"
  image_tag_mutability = "MUTABLE"                       # staging; prod would be IMMUTABLE
  image_scanning_configuration { scan_on_push = true }
}

# ----- ECS cluster -----
resource "aws_ecs_cluster" "main" { name = "shoplet-staging-fargate" }

# ----- CloudWatch log groups (one per bundle) -----
resource "aws_cloudwatch_log_group" "api_svc"   { name = "/ecs/shoplet/api_svc";   retention_in_days = 14 }
# worker_fn is Lambda; its log group is /aws/lambda/<func>, auto-created.
# report_b is Batch; logs go to /aws/batch/job, auto-created.

# ----- bundles.api_svc (modules: [api], shape: long_running, on: fargate) -----
resource "aws_ecs_task_definition" "api_svc" {
  family                   = "api_svc"
  requires_compatibilities = ["FARGATE"]
  network_mode             = "awsvpc"
  cpu                      = "512"                       # from yaml size: "0.5vCPU/1GB"
  memory                   = "1024"
  execution_role_arn       = aws_iam_role.task_execution.arn
  task_role_arn            = aws_iam_role.api_svc_task.arn
  container_definitions = jsonencode([{
    name      = "app"
    image     = "${aws_ecr_repository.app.repository_url}:latest"
    command   = ["--module=api"]                         # dispatcher: run module `api`
    essential = true
    portMappings = [{ containerPort = 8080, protocol = "tcp" }]
    environment = [
      { name = "UPLOADS_BUCKET", value = aws_s3_bucket.uploads.id },
      { name = "JOBS_QUEUE_URL", value = aws_sqs_queue.jobs.id },
      { name = "APP_DB_DATABASE_URL_HOST", value = aws_db_instance.app_db.address },
      # SUPEUX_RPC_PEERS injected: worker_fn is in a different bundle (Lambda).
      { name = "SUPEUX_RPC_BUNDLE", value = "api_svc" },
      { name = "SUPEUX_RPC_SELF",   value = "api" },
      { name = "SUPEUX_RPC_PEERS",  value = jsonencode({
          worker = { mode = "lambda", function_url = aws_lambda_function_url.worker_fn.function_url }
      }) }
    ]
    secrets = [
      # Resolved at container start; the secret value never appears in plaintext HCL.
      { name = "APP_DB_DATABASE_URL", valueFrom = aws_db_instance.app_db.master_user_secret[0].secret_arn }
    ]
    logConfiguration = {
      logDriver = "awslogs"
      options   = {
        awslogs-group         = aws_cloudwatch_log_group.api_svc.name
        awslogs-region        = var.region
        awslogs-stream-prefix = "api"
      }
    }
  }])
}

resource "aws_ecs_service" "api_svc" {
  name            = "api_svc"
  cluster         = aws_ecs_cluster.main.id
  task_definition = aws_ecs_task_definition.api_svc.arn
  desired_count   = 1
  launch_type     = "FARGATE"
  network_configuration {
    subnets          = [for s in aws_subnet.private : s.id]
    security_groups  = [aws_security_group.api_svc.id]
    assign_public_ip = false                             # ALB fronts public traffic
  }
  load_balancer {
    target_group_arn = aws_lb_target_group.api.arn
    container_name   = "app"
    container_port   = 8080
  }
  depends_on = [aws_lb_listener.api_http]
}

# ----- bundles.worker_fn (modules: [worker], shape: function, on: lambda) -----
# Same image; Lambda packaging instead of Fargate. Function URL for inter-module
# calls from api_svc. SQS event source mapping wired to `jobs` queue.
resource "aws_lambda_function" "worker_fn" {
  function_name = "shoplet-worker"
  role          = aws_iam_role.worker_fn_task.arn
  package_type  = "Image"
  image_uri     = "${aws_ecr_repository.app.repository_url}:latest"
  image_config { command = ["--module=worker"] }
  memory_size   = 1024                                   # from yaml size: "1024MB"
  timeout       = 300
  environment {
    variables = {
      UPLOADS_BUCKET             = aws_s3_bucket.uploads.id
      JOBS_QUEUE_URL             = aws_sqs_queue.jobs.id
      APP_DB_DATABASE_URL_HOST   = aws_db_instance.app_db.address
      APP_DB_DATABASE_URL_SECRET = aws_db_instance.app_db.master_user_secret[0].secret_arn
      SUPEUX_RPC_BUNDLE          = "worker_fn"
      SUPEUX_RPC_SELF            = "worker"
      SUPEUX_RPC_PEERS           = jsonencode({})        # worker calls no peers in this app
    }
  }
}

resource "aws_lambda_function_url" "worker_fn" {
  function_name      = aws_lambda_function.worker_fn.function_name
  authorization_type = "AWS_IAM"                         # api_svc calls this IAM-signed
}

# SQS → Lambda event source mapping. Only emitted because worker is shape:function.
# A Fargate worker would poll SQS itself in user code instead.
resource "aws_lambda_event_source_mapping" "worker_fn_jobs" {
  event_source_arn = aws_sqs_queue.jobs.arn
  function_name    = aws_lambda_function.worker_fn.function_name
  batch_size       = 10
}

# Cron trigger from worker.triggers[1]: schedule "0 2 * * *" UTC.
resource "aws_scheduler_schedule" "worker_fn_nightly_cleanup" {
  name                         = "shoplet-worker-nightly_cleanup"
  group_name                   = "default"
  schedule_expression          = "cron(0 2 * * ? *)"     # EventBridge cron dialect
  schedule_expression_timezone = "UTC"
  target {
    arn      = aws_lambda_function.worker_fn.arn
    role_arn = aws_iam_role.scheduler.arn
    input    = jsonencode({ trigger = "cron", name = "nightly_cleanup" })
  }
  flexible_time_window { mode = "OFF" }
}

# ----- bundles.report_b (modules: [report], shape: batch, on: batch) -----
# Omitted for space — emits aws_batch_job_definition + aws_batch_job_queue + compute environment.
# IAM same-shape as worker_fn_task (S3+RDS on app_db + archives).

# ----- ALB for api_svc (module api has expose.public=true) -----
resource "aws_lb" "api" {
  name               = "shoplet-api-staging"
  internal           = false
  load_balancer_type = "application"
  security_groups    = [aws_security_group.alb.id]
  subnets            = [for s in aws_subnet.public : s.id]
}

resource "aws_lb_target_group" "api" {
  name        = "shoplet-api-tg"
  port        = 8080
  protocol    = "HTTP"
  vpc_id      = aws_vpc.main.id
  target_type = "ip"                                     # required for Fargate awsvpc
  health_check { path = "/health"; matcher = "200-299" }
}

resource "aws_lb_listener" "api_http" {
  load_balancer_arn = aws_lb.api.arn
  port              = 80
  protocol          = "HTTP"
  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.api.arn
  }
}
```

### 2b. `network.tf`

```hcl
# ----- VPC + subnets + NAT (auto-derived; emitted because cloud bundles exist) -----
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"                   # supeux convention; future yaml override
  enable_dns_hostnames = true
  enable_dns_support   = true
}

data "aws_availability_zones" "available" { state = "available" }

resource "aws_subnet" "public" {
  for_each                = { for i, az in slice(data.aws_availability_zones.available.names, 0, 2) : az => i }
  vpc_id                  = aws_vpc.main.id
  cidr_block              = "10.0.${each.value}.0/24"
  availability_zone       = each.key
  map_public_ip_on_launch = true
}

resource "aws_subnet" "private" {
  for_each          = { for i, az in slice(data.aws_availability_zones.available.names, 0, 2) : az => i + 10 }
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.${each.value}.0/24"
  availability_zone = each.key
}

resource "aws_internet_gateway" "main" { vpc_id = aws_vpc.main.id }
resource "aws_eip" "nat" { domain = "vpc" }
resource "aws_nat_gateway" "main" {
  allocation_id = aws_eip.nat.id
  subnet_id     = values(aws_subnet.public)[0].id
}
# Route tables omitted: public → IGW, private → NAT.

# ----- Security groups (auto-derived from `uses:` graph) -----
# For each bundle: one SG. For each cloud resource: one SG with ingress restricted
# to the SGs of bundles whose modules use that resource. ALB has its own.

resource "aws_security_group" "alb" {
  name   = "alb-staging"
  vpc_id = aws_vpc.main.id
  ingress { from_port = 80; to_port = 80; protocol = "tcp"; cidr_blocks = ["0.0.0.0/0"] }
  egress  { from_port = 0;  to_port = 0;  protocol = "-1"; cidr_blocks = ["0.0.0.0/0"] }
}

resource "aws_security_group" "api_svc" {
  name   = "api-svc-staging"
  vpc_id = aws_vpc.main.id
  ingress { from_port = 8080; to_port = 8080; protocol = "tcp"; security_groups = [aws_security_group.alb.id] }
  egress  { from_port = 0; to_port = 0; protocol = "-1"; cidr_blocks = ["0.0.0.0/0"] }
}

# worker_fn is Lambda — uses Lambda-managed SG inside its own VPC config; not emitted here.

resource "aws_security_group" "app_db" {
  name   = "app-db-staging"
  vpc_id = aws_vpc.main.id
  ingress {
    from_port       = 5432
    to_port         = 5432
    protocol        = "tcp"
    security_groups = [aws_security_group.api_svc.id]    # worker_fn would need VPC config + this SG added
  }
}

# Bucket + queue have no SG (gated by IAM, public AWS API endpoints).
```

## 3. Sample compose — `dev-local` target

`infra/dev-local/generated/docker-compose.generated.yaml`:

```yaml
# === Generated by supeux compile --target=dev-local. DO NOT EDIT. ===
# Composition: every resource → local OSS engine.
# Placement: monolith bundle → compose. ONE container, three modules co-located.

version: "3.9"

services:
  # ----- local stand-ins for resources (composition: oss-local) -----
  postgres:
    image: postgres:16                          # matches app_db.version
    environment:
      POSTGRES_USER: app
      POSTGRES_PASSWORD: app                     # dev-only static creds
      POSTGRES_DB: app
    ports: ["5432:5432"]
    volumes: ["pg_data:/var/lib/postgresql/data"]

  minio:
    image: minio/minio:latest                    # stands in for bucket=uploads (+ archives)
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    ports: ["9000:9000", "9001:9001"]
    volumes: ["minio_data:/data"]

  minio_init:                                    # one-shot bucket creation
    image: minio/mc:latest
    depends_on: [minio]
    entrypoint: >
      sh -c "until mc alias set local http://minio:9000 minioadmin minioadmin; do sleep 1; done;
             mc mb -p local/uploads || true;
             mc mb -p local/archives || true"

  elasticmq:
    image: softwaremill/elasticmq-native:latest  # SQS-compatible
    ports: ["9324:9324"]

  # ----- bundle: monolith (modules: api, worker, report) -----
  # ONE container running ALL three modules. The image dispatcher reads
  # --modules=a,b,c and starts each module in-process (goroutine / thread / task).
  # SUPEUX_RPC_PEERS marks every sibling as inproc — no network calls.
  monolith:
    build: .                                     # repo root; same image as cloud
    command: ["--modules=api,worker,report"]
    ports: ["8080:8080"]                         # module api still expose:8080
    depends_on: [postgres, minio, minio_init, elasticmq]
    environment:
      # Tier-0 endpoint-URL swap (supeux_abstraction_v4 §4)
      S3_ENDPOINT_URL: http://minio:9000
      SQS_ENDPOINT_URL: http://elasticmq:9324
      AWS_ACCESS_KEY_ID: minioadmin
      AWS_SECRET_ACCESS_KEY: minioadmin
      AWS_REGION: us-east-1
      # Canonical env-var names (identical to cloud — that's the contract)
      UPLOADS_BUCKET: uploads
      ARCHIVES_BUCKET: archives
      JOBS_QUEUE_URL: http://elasticmq:9324/queue/jobs
      APP_DB_DATABASE_URL: postgres://app:app@postgres:5432/app
      # RPC dispatch — everyone is inproc in the monolith
      SUPEUX_RPC_BUNDLE: monolith
      SUPEUX_RPC_PEERS: '{"api":{"mode":"inproc"},"worker":{"mode":"inproc"},"report":{"mode":"inproc"}}'

volumes:
  pg_data:
  minio_data:
```

Divergences from the HCL version:
- No IAM, no VPC, no SG, no ALB.
- One process, three modules. Cloud has three separate workloads (Fargate + Lambda + Batch).
- Endpoint-URL env vars are the runtime switch (Tier-0 contract).
- `APP_DB_DATABASE_URL` is literal, not a Secrets Manager indirection.
- No ECR. Image built locally by `docker compose build`, never pushed.
- No log group. `docker logs` reads stdout/stderr.

## 4. `hybrid-dev` delta — mixed composition

Local `monolith` container talking to real S3 + real Bedrock; Postgres stays local. This is the thesis dimension-3 canonical case (mixing is first-class).

### 4a. Compose env-var diff vs `dev-local`

```diff
   monolith:
     environment:
-      S3_ENDPOINT_URL: http://minio:9000
-      AWS_ACCESS_KEY_ID: minioadmin
-      AWS_SECRET_ACCESS_KEY: minioadmin
+      # No S3 endpoint override — SDK hits real AWS.
+      AWS_PROFILE: hybrid-dev                  # creds from ~/.aws/credentials
       AWS_REGION: us-east-1
       SQS_ENDPOINT_URL: http://elasticmq:9324  # jobs stays local
-      UPLOADS_BUCKET: uploads
+      UPLOADS_BUCKET: shoplet-uploads-hybrid-dev   # real S3 bucket name
       APP_DB_DATABASE_URL: postgres://app:app@postgres:5432/app
+    volumes:
+      - ~/.aws:/root/.aws:ro                   # mount real AWS creds read-only
```

Compose-file shape changes:
- `minio` and `minio_init` services and `minio_data` volume removed (uploads now cloud-managed).
- `postgres`, `elasticmq`, `pg_data` kept (composition: oss-local for `jobs` and `app_db`).
- `monolith` env block modified per diff.

### 4b. Minimal `infra/hybrid-dev/generated/main.tf`

```hcl
# === Why this file exists when monolith is LOCAL: ===
# Resources with composition=cloud-managed (uploads, llm in this strategy) must
# be provisioned in AWS even though no compute lives in cloud.

resource "aws_s3_bucket" "uploads" {
  bucket = "shoplet-uploads-hybrid-dev"
}

# llm (cloud_only) — Bedrock isn't a Terraform-provisioned resource per se; the
# emitter records the model ID in the env var instead. The cloud_only registry
# in supeux knows whether each cloud_only.type emits HCL or just env config.

# No VPC, no IAM role, no ECS — the human's AWS profile carries the creds.
# A production hybrid-dev would emit aws_iam_user + access_key here; current
# convention says hybrid-dev is for short-lived debugging, use the human's creds.

output "uploads_bucket" { value = aws_s3_bucket.uploads.id }
```

The compose env var `UPLOADS_BUCKET=shoplet-uploads-hybrid-dev` comes from the supeux naming convention `<app>-<resource>-<target>`. Drift between this convention and reality (manual rename in cloud) requires `supeux refresh` to reconcile.
